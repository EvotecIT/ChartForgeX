    <h2>Site shell and chart output are separate on purpose.</h2>
    <p>The dedicated site owns the Evotec-aligned chrome, navigation, search, docs, and browsing workflow. ChartForgeX owns the rendered examples, the source code that builds them, and the visual fidelity of each SVG, PNG, and HTML case.</p>

<section class="section-grid">
  <article>
    <p class="eyebrow">Pilot contract</p>
    <h2>Dedicated site, project-owned source</h2>
    <p>This site is the first dedicated project-site pilot for visual-heavy Evotec projects. The central Evotec hub keeps the project summary, while this subdomain owns the gallery, documentation, and rendered examples.</p>
  </article>
  <article>
    <p class="eyebrow">Source-linked demos</p>
    <h2>Every visual points back to code</h2>
    <p>Example cards should show the rendered output, the HTML/SVG/PNG artifacts, and the exact source file or method that produced the case so a developer can copy it and build from there.</p>
  </article>
  <article>
    <p class="eyebrow">Reusable pattern</p>
    <h2>Project flavor, shared foundation</h2>
    <p>Navigation, metadata, docs, sitemap, search, and quality gates follow PowerForge conventions. The visual identity belongs to ChartForgeX through the local theme and gallery pages.</p>
  </article>
