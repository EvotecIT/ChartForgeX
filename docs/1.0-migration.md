# ChartForgeX 1.0 Migration

ChartForgeX 1.0 removes pre-release duplicate construction and styling surfaces. The migration is intentionally breaking: there is one public model for each reusable concept, and hosts keep product-specific data outside the renderer.

## Construction and data

`ChartForgeX.Simple` and its deferred chart definitions have been removed. Build native `Chart` objects directly, or map host records with `ChartDataset<T>` and the typed `AddLine`, `AddBar`, `AddArea`, `AddScatter`, and `AddHistogram` overloads.

```csharp
using ChartForgeX.Core;
using ChartForgeX.Data;

var rows = ChartDataset<Sample>.From(sourceRows)
    .Filter(row => row.Enabled)
    .SortBy(row => row.Timestamp);

var chart = Chart.Create()
    .AddLine("Latency", rows, row => row.Timestamp.ToOADate(), row => row.LatencyMs)
    .WithXAxisScale(ChartScaleKind.Time)
    .WithYAxisScale(ChartScaleKind.Logarithmic);
```

Use `ChartGrid.FromFacets(...)` for deterministic small multiples. Product DTOs, data collection, filters, and dashboard shells remain in the host.

## Consolidated types

| Before 1.0 | 1.0 surface |
| --- | --- |
| `ChartTextStyle` | `TextStyleOverride` for optional chart-role overrides; `TextStyle` for a complete resolved style |
| `VisualTextAlignment`, `VisualCanvasTextAlignment` | `TextAlignment` |
| `ChartGridPanelFit`, `VisualGridPanelFit` | `VisualPanelFit` |
| `VisualArtifactRect` | `ChartRect` |
| `TopologyDirection`, `FlowArtifactConnectorDirection`, `WardleyMapFlow` | `VisualLinkDirection` |
| `System.Drawing.Color`, `PointF`, `RectangleF` in consuming hosts | `ChartColor`, `ChartPoint`, `ChartRect` |

No compatibility aliases are provided. This keeps old names from becoming a second public API that must be maintained after 1.0.

## Axes and scales

`ChartAxis` is the source of truth for explicit or automatic bounds, tick count, label density, label angle, visibility, formatting, and scale selection. Configure it through `ConfigureXAxis`, `ConfigureYAxis`, or `ConfigureSecondaryYAxis`.

Supported scale kinds are `Linear`, `Logarithmic`, `SymmetricLogarithmic`, `Time`, `Category`, and `Band`. SVG and PNG use the same scale transform and tick model. Logarithmic axes reject non-positive bounds and data instead of producing invalid output.

## Typography

`FontSpec`, `TextStyle`, `TextStyleOverride`, `TextAlignment`, and `TextLayoutEngine` form the shared dependency-free typography contract. Charts, topology, visual blocks, visual canvas, and `ImageComposition` resolve text through this layer. Raster output can select TrueType files or collections without relying on `System.Drawing`.

## Design and accessibility

`VisualDesignTokens` applies product-neutral colors, typography, radii, and stroke width across `ChartTheme`, `VisualCanvasTheme`, and `TopologyTheme`. Hosts can use one token set for charts, topology diagrams, reports, and wallpaper composition.

Charts, topology, and visual canvases expose `VisualAccessibility` through `WithAccessibility(...)` and `AsDecorative()`. Capable SVG renderers emit a title, description, language, role, and ARIA metadata; decorative visuals are hidden from assistive technology.

## Responsive composition

`VisualCanvas.WithResponsiveLayout(designWidth, designHeight, fit)` separates layer coordinates from output dimensions. `Contain`, `Cover`, and `Stretch` use the same design-space contract in SVG and PNG, so a host can reuse one composition for multiple wallpaper, report-cover, or social-image sizes.

## Topology mapping

`TopologyChart.FromData<TNode, TEdge>(...)` maps product-owned node and edge records into the product-neutral topology model. The mapper preserves input order and rejects duplicate node ids, duplicate edge ids, and dangling endpoints before rendering.

## Raster input and dependencies

The core ChartForgeX package has no runtime package dependencies. Its raster input supports PNG, GIF, JPEG, BMP, PPM, and uncompressed RGB TIFF through dependency-free readers. GIF input uses the first frame; animated GIF and APNG output remain separate deterministic export paths.

ImagePlayground now consumes native ChartForgeX charts instead of maintaining its own chart command DTOs and renderer choices. PowerBGInfo uses ChartForgeX colors, geometry, typography, composition, and raster input instead of `System.Drawing`.
